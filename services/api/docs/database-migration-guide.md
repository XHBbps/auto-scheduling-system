# 数据库迁移指南

## 目标
- 统一使用 **Alembic** 管理数据库结构变更
- 默认不再依赖运行时自动 `create_all`
- 避免不同环境因为手工建表或旧表结构导致行为不一致

## 当前迁移概况
- `20260320_0001`：初始化基础 schema
- `20260320_0002`：补充 BOM 补数相关表结构

## 本地使用步骤

### 1. 查看当前迁移状态
```bash
alembic current
alembic heads
```

### 2. 升级到最新版本
```bash
alembic upgrade head
```

### 3. 新增迁移
在模型变更完成后执行：
```bash
alembic revision --autogenerate -m "描述本次结构变更"
```

生成迁移文件后，必须人工检查：
- 是否只包含本次预期变更
- 是否误删已有表或字段
- 是否包含数据库默认值、索引、约束等必要调整

## Docker 环境说明

如果通过 Docker / Compose 启动后端服务，建议在服务启动前或启动流程中显式执行：
```bash
alembic upgrade head
```

这样可以保证容器环境与本地环境使用同一套迁移基线。

## 注意事项
- **已有旧库但没有迁移历史**：先确认表结构是否与基线一致，再决定使用 `alembic stamp` 还是重新建库
- **全新环境**：直接执行 `alembic upgrade head`
- 不建议继续依赖运行时自动建表作为正式方案
- 每次结构变更都应同步更新对应文档与验证记录
