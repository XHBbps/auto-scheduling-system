## 2. `.env` 需要填写哪些项

## 2.1 观远 BI（销售计划）

对应代码：

- `app/integration/guandata_client.py`
- `app/sync/sales_plan_sync.py`

需要填写：

```env
GUANDATA_BASE_URL=
GUANDATA_DOMAIN=
GUANDATA_LOGIN_ID=
GUANDATA_PASSWORD=
GUANDATA_DS_ID=
```

### 字段说明

| 变量 | 含义 | 从哪里拿 |
|---|---|---|
| `GUANDATA_BASE_URL` | 观远接口根地址 | 观远接口文档 / 已有系统调用配置 |
| `GUANDATA_DOMAIN` | 登录域名参数 | 观远账号所属 domain |
| `GUANDATA_LOGIN_ID` | 登录账号 | 用于调用接口的账号 |
| `GUANDATA_PASSWORD` | 登录密码 | 对应账号密码 |
| `GUANDATA_DS_ID` | 销售计划数据集 ID | 观远数据集配置页 / 既有接口文档 |

### 当前代码里的调用方式

1. 登录接口：

```text
POST {GUANDATA_BASE_URL}/public-api/sign-in
```

请求体核心字段：

- `domain`
- `loginId`
- `password`

2. 取数据接口：

```text
POST {GUANDATA_BASE_URL}/public-api/data-source/{GUANDATA_DS_ID}/data
```

请求体核心字段：

- `offset`
- `limit`

### 联调时重点核对

- `dsId` 是否真的是销售计划数据集
- 返回的 `preview` 数组字段顺序是否与代码假设一致
- 是否确实存在：
  - `sap_code`
  - `sap_line_no`
  - `detail_id`
  - `material_no`
  - `confirmed_delivery_date`
  - `delivery_plant`

## 2.2 SAP BOM

对应代码：

- `app/integration/sap_bom_client.py`
- `app/sync/bom_sync.py`

需要填写：

```env
SAP_BOM_BASE_URL=
```

### 字段说明

| 变量 | 含义 | 从哪里拿 |
|---|---|---|
| `SAP_BOM_BASE_URL` | SAP BOM 查询接口完整地址 | SAP 接口文档 / 中台接口配置 |

### 当前代码里的调用方式

请求方式：

```text
POST {SAP_BOM_BASE_URL}
```

请求体核心字段：

- `IS_MTNRV`：整机物料号
- `IS_WERKS`：工厂

### 联调时重点核对

- 返回结构是否有：
  - `ES_RET`
  - `LT_BOM001.item`
- 成功/失败是否仍按 `ES_RET.CODE`
- 顶层冗余 BOM 行是否仍满足：
  - `bom_component_no == machine_material_no`

## 2.3 飞书应用（生产订单 + 研究所）

对应代码：

- `app/integration/feishu_client.py`
- `app/integration/feishu_field_maps.py`
- `app/sync/production_order_sync.py`
- `app/sync/research_sync.py`

需要填写：

```env
FEISHU_APP_ID=
FEISHU_APP_SECRET=
FEISHU_PRODUCTION_APP_TOKEN=
FEISHU_PRODUCTION_TABLE_ID=
FEISHU_RESEARCH_APP_TOKEN=
FEISHU_RESEARCH_TABLE_ID=
```

### 字段说明

| 变量 | 含义 | 从哪里拿 |
|---|---|---|
| `FEISHU_APP_ID` | 飞书自建应用 App ID | 飞书开放平台应用配置 |
| `FEISHU_APP_SECRET` | 飞书自建应用 App Secret | 飞书开放平台应用配置 |
| `FEISHU_PRODUCTION_APP_TOKEN` | 生产订单多维表格 app token | 飞书多维表格链接 / API 配置 |
| `FEISHU_PRODUCTION_TABLE_ID` | 生产订单表 ID | 多维表格对应表的 table id |
| `FEISHU_RESEARCH_APP_TOKEN` | 研究所多维表格 app token | 飞书多维表格链接 / API 配置 |
| `FEISHU_RESEARCH_TABLE_ID` | 研究所表 ID | 多维表格对应表的 table id |

### 当前代码里的调用方式

1. 先获取 tenant token：

```text
POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal
```

2. 再查记录：

```text
POST https://open.feishu.cn/open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records/search
```

### 联调时重点核对

- `生产工厂`
- `加工部门`
- `投产时间`
- `完工时间`
- `订货数量`
- `生产订单状态`
- `销售订单号`
- `最后更新时间`

#### 研究所表

字段至少要与代码映射一致：

- `订单编号`
- `明细ID`
- `明细-物料编号`
- `发图时间（研究所）`
- `明细-产品型号`
- `产品大系列`
- `明细-数量`
- `报检日期`
- `定制编号`

## 3. 建议填写顺序

建议不要一次性乱填，按这个顺序来：

1. 先确认并填写 `DATABASE_URL`
2. 再填写观远
3. 再填写飞书
4. 最后填写 SAP BOM

原因：

- 销售计划和研究所联调优先级最高
- 研究所同步后要验证发图状态自动回填
- BOM 同步依赖物料号 + 工厂，适合放后面

---

## 4. 填写后如何快速验证

填写完成后，按下面顺序在 Swagger 验证：

1. `POST /api/admin/sync/sales-plan`
2. `GET /api/data/sales-plan-orders`
3. `POST /api/admin/sync/research`
4. `GET /api/data/machine-cycle-history`
5. 再次 `GET /api/data/sales-plan-orders`，确认 `drawing_released`
6. `POST /api/admin/sync/production-orders`
7. `POST /api/admin/sync/bom`

## 5. 常见填写错误

### 观远

- base_url 填成页面地址而不是 API 根地址
- `dsId` 填错数据集
- domain 漏填

### 飞书

- app token 和 table id 混淆
- 用了错误的应用 app_id / app_secret
- 表字段中文名与代码配置不一致

### SAP

- 把接口文档地址当成真实调用地址
- 返回结构与代码假设不一致但未先验证

## 7. 2026-03-18 真实接口差异补充

### 飞书 Bitable

本轮真实联调确认：

1. `records/search` 当前**不要继续依赖**原设计里的 `field_names`
   - 真实请求会返回 `FieldNameNotFound`

2. 原设计里的 `filter.conditions[].field_name` 当前也**不能直接用**
   - 真实请求会返回 `InvalidFilter`

3. `page_token` 必须放在 **query string**
   - 如果误放 body，会重复拿到第一页数据

4. 研究所真实字段名应使用：
   - `报检时间`
   - 不再使用旧文档中的 `报检日期`

当前代码已按“**分页走服务端，字段过滤/增量过滤走本地**”修正。

### SAP BOM

本轮真实联调确认：

1. SAP 返回的以下字段可能是数值型而不是字符串：
   - 整机物料号
   - 工厂
   - 父项物料号
   - 子项物料号

2. 这些字段在写入 PostgreSQL 前必须统一转字符串
   - 否则会触发 `asyncpg expected str, got int`
