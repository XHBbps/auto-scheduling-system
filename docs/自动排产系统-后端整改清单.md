# 自动排产系统-后端整改清单

> 更新日期：2026-03-26
> 用途：供后端人员并行落地整改项。
> 维护规则：本文件只能勾选复选框，不允许修改正文内容；完成事实请回写 `docs/progress/自动排产系统-后端.md`。

## 使用说明
- [x] 阅读并确认本清单只允许勾选，不允许改正文。
- [x] 开始执行前先对照 `docs/Agent.md`、`docs/progress/自动排产系统-后端.md` 与 `docs/openapi.json`，确认当前口径。
- [x] 每完成一项后，仅更新勾选状态，并将完成事实、验证方式、风险变化写入后端 progress 文档。

## P0（必须优先收口）
- [x] 明确并收口 plant 维度：在 BOM、排产、快照、结果查询、导出链路中统一是否必须按 plant 维度处理；验收标准：同一订单/物料/结果在不同接口与任务链路中的 plant 口径一致，不再出现“有的链路有 plant、有的链路没有 plant”的半状态。
- [x] 如 plant 为必填维度，则同步收口数据源、模型、服务逻辑、查询条件与 OpenAPI；验收标准：数据库字段、接口参数、业务逻辑与文档说明完全一致，前后端不再靠猜测补字段。
- [x] 拆分后台任务长事务：缩小 worker / scheduler / 同步任务中的事务边界，避免单次任务占用事务过久；验收标准：长任务失败时回滚范围可控，锁冲突与阻塞风险下降。
- [x] 优化导出链路：避免全量结果在 Python 内存中一次性组装后再导出；验收标准：导出流程在大数据量下内存曲线可控，失败路径与重试策略清晰。
- [x] 明确 PostgreSQL 模糊搜索策略：针对 `%keyword%`、排序、分页链路制定索引/查询方案；验收标准：高频模糊搜索接口有明确优化策略，不再默认接受全表扫描。
- [x] 使用真实 PostgreSQL 环境补齐集成验证：重点验证 JSONB、锁、队列、重试、分页与事务边界；验收标准：不只在 SQLite/轻量场景成立，关键链路在真实 PostgreSQL 上也可解释、可复现。

## P1（高价值增强项）
- [x] 收口 snapshot seed / batch refresh / rebuild 热点：继续排查快照初始化、批量刷新、重建链路中的重复查询与重复组装；验收标准：热点链路的主要 SQL 与对象装载路径可说明、可优化。
- [x] 继续清理导出/BOM/snapshot/可观测性链路剩余 SQL 热点；验收标准：P0 之外的热点有明确清单、优先级与验证结论。
- [x] RBAC 从 `require_admin_role` 升级到权限点控制预案；验收标准：角色与权限模型具备可扩展路径，不会把后续权限设计锁死在单一 admin 角色上。
- [x] OpenAPI 收口：补全 header/query/path/body 参数中文说明、响应结构、鉴权说明、错误码示例；验收标准：Swagger / OpenAPI 可直接作为联调依据，而非仅能“看个大概”。
- [x] 强化 session 安全：补充 session 失效、并发登录、密码变更后会话处理、停用用户回收等边界策略；验收标准：会话安全口径完整，行为可测试、可文档化。
- [x] worker / scheduler / 后台任务恢复测试补齐；验收标准：服务重启、任务中断、重试、重复入队、幂等场景有明确验证结果。

## P2（结构与语义治理）
- [ ] 继续拆分超大文件：优先处理编排、导出、同步、查询、快照相关的大体量 service/repository/router 文件；验收标准：主文件职责收敛，辅助逻辑与编排逻辑分层清楚。
- [ ] 复核 `pending_drawing` 等复用状态语义；验收标准：状态值在排产、同步、查询、展示上的含义一致，不再一值多义。
- [ ] 复核机加周期、装配周期、触发日期等排产语义边界；验收标准：核心排产字段的业务含义有统一口径，避免同字段承载多重含义。
- [x] 强化可观测性：补充后台任务、导出、同步、快照链路的日志、指标与异常分类；验收标准：线上排查可定位到任务、阶段、SQL 热点与失败原因。
- [x] 为库存/产能实时校验预留统一读层；验收标准：未来扩展库存、产能、约束校验时，不必重写现有查询入口。

## P3（数据库与一致性治理）
- [ ] 复核约束、软关联与一致性检查：明确哪些规则依赖数据库约束，哪些依赖业务层兜底；验收标准：关键数据一致性有清晰守护层，不再完全靠人工约定。
- [ ] 减少 Python 侧全量聚合：尽量把过滤、聚合、分页前置到数据库侧或分段处理；验收标准：关键慢链路能解释为何在数据库做、为何在应用层做，并有可度量收益。
- [x] 沉淀 worker / scheduler / snapshot 运维说明；验收标准：后端同学接手任务系统时可快速理解启动方式、观察指标与故障处理流程。

## P4（生产化长期项）
- [ ] 持续推进更完整的生产化能力：包括限流、降级、告警、容量评估、异常恢复演练；验收标准：系统从“可运行”逐步走向“可长期稳定运行”。
- [ ] 整改完成事实按阶段回写后端 progress；验收标准：每次勾选都有对应事实留痕，progress 文档可反映真实完成度。
