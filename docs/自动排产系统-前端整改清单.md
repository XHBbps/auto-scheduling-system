## 使用说明
- [x] 阅读并确认本清单只允许勾选，不允许改正文。
- [x] 开始执行前先对照 `docs/Agent.md` 与 `docs/progress/自动排产系统-前端.md`，确认当前优先级与依赖关系。
- [x] 每完成一项后，仅更新勾选状态，并将完成事实、验证方式、阻塞点写入前端 progress 文档。

## P0（必须优先收口）
- [x] 认证主链路前端回归补齐：覆盖登录、路由守卫、刷新保活、退出登录、未登录拦截、403/401 跳转等主链路场景；验收标准：形成可重复执行的回归用例，关键认证页面与关键入口无手工口径分歧。
- [x] Dashboard 数据获取链路收口：清理长期依赖“大范围全量拉取 + 前端本地聚合”的兜底方案，明确首页卡片、图表、摘要数据的获取边界；验收标准：Dashboard 不再依赖不可控的大批量前端聚合，异常与空态路径可解释。
- [x] Dashboard 中文文案与口径统一：统一日期、状态、统计字段、按钮文案、错误提示与空态描述；验收标准：同一业务对象在 Dashboard 内外展示口径一致，不再出现混杂命名与半成品提示语。
- [x] 机台/零件列表加载体验补齐：统一 loading、empty、error、disabled、权限受限等状态表现；验收标准：列表页在各种状态下均有明确反馈，且用户不会误判为“卡死”或“无数据”。
- [x] 导出交互体验收口：统一导出前确认、导出中状态、导出成功/失败 toast、下载入口与异常提示；验收标准：导出链路从发起到结果反馈闭环清晰，不再出现无反馈或重复点击问题。
- [x] 高复杂度页面继续拆分：优先拆分 `ScheduleDashboardPage.vue`、`WorkCalendarPage.vue`、`MachineScheduleListPage.vue`、`PartScheduleListPage.vue`；验收标准：页面主文件职责收敛，状态管理与展示组件边界清晰，可维护性明显提升。

## P1（高价值增强项）
- [x] 列表页复用层继续扩展：扩展 `useServerTableQuery.ts`、`useTableSort.ts`、`AppTableState.vue`、`useTableFeedbackState.ts` 的覆盖范围；验收标准：查询、排序、分页、状态反馈的实现方式进一步统一，重复代码显著减少。
- [x] 用户管理页面体验收口：统一筛选、弹窗、禁用状态、重置密码、角色编辑等交互细节；验收标准：用户管理操作反馈清晰，危险操作有明确确认与结果提示。
- [x] 同步控制台状态展示优化：统一任务状态标签、刷新时机、失败原因展示与重试入口提示；验收标准：同步台可以直接解释“正在执行/已排队/失败/无需执行”等状态，不依赖猜测。
- [x] 查询参数与排序字段命名统一：统一前端内部参数、URL 查询参数、接口映射字段名称；验收标准：同类列表页不再出现一页一套命名方式，排查问题时可快速定位。
- [x] 全局文案口径治理：统一日期格式、状态枚举、字段中文名、按钮标签、表头命名；验收标准：同一领域术语全站保持一致，减少培训与误解成本。
- [x] 继续拆分基线配置大页：拆分 `PartCycleBaselinePage.vue`、`MachineCycleBaselinePage.vue`；验收标准：查询区、表格区、弹窗区、业务编排层分离清楚，后续修改不再牵一发动全身。

## P2（质量与性能补强）
- [x] 前端自动化测试补强：覆盖 Dashboard、列表页、用户管理、同步控制台等高价值场景；验收标准：关键页面至少具备基础回归保护，能支撑并行开发后的快速回归。
- [x] 表格/图表/日历性能观测补齐：记录慢渲染、重复请求、无效重算、长列表交互卡顿点；验收标准：能明确指出瓶颈页面、瓶颈组件与触发条件，而不是仅凭体感优化。
- [x] 构建产物 chunk 继续观测：持续观察图表、日期组件、重页面拆分后的 chunk 变化；验收标准：构建产物无明显失控增长，关键大依赖的引入与拆分可追踪。
- [x] 组件与状态管理范式收敛：统一“页面编排层 + 业务 composable + 展示组件”模式；验收标准：新增页面与重构页面遵循同一组织方式，减少再次形成超大页面文件。
- [x] 统一确认框/禁用态/toast/loading 交互基线；验收标准：用户在不同页面面对相似操作时获得一致反馈，不再出现一页一个习惯。

## P3（中长期预埋）
- [x] 预留前端 RBAC 接入点：在路由、按钮级能力、页面级能力上预留更细粒度权限扩展位；验收标准：后端 RBAC 升级后无需大范围推翻前端结构。
- [x] 状态展示组件收敛：提炼通用状态标签、结果提示、空态/异常态组件；验收标准：状态展示可复用，减少页面各自维护状态样式与文案。
- [x] 整改完成事实按阶段回写前端 progress；验收标准：每次勾选都有对应事实留痕，progress 文档可反映真实完成度。
