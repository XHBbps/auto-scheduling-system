# 前端性能观测

## 当前覆盖范围

- `dashboard:reloadOverview`：Dashboard 总览接口刷新
- `dashboard:initCharts`：Dashboard 图表初始化
- `workCalendar:fetchDistributionData`：日历月分布加载
- `workCalendar:openDayDetail`：日历单日详情加载
- `machineScheduleList:fetchData`：整机排产列表查询
- `machineScheduleList:fetchProductSeriesOptions`：整机排产筛选选项加载
- `partScheduleList:fetchData`：零件排产列表查询
- `partScheduleList:fetchAssemblyOptions`：零件排产筛选选项加载
- `issueManagement:fetchIssueTable`：异常管理列表查询
- `machineCycleHistory:fetchHistoryTable`：整机周期历史列表查询
- `productionOrderData:fetchProductionOrderTable`：生产订单历史列表查询
- `salesPlanData:fetchSalesPlanTable`：销售计划源数据列表查询
- `syncLogList:fetchSyncLogTable`：同步日志列表查询

## 观测方式

- 浏览器端性能指标统一写入 `window.__SOUL_PERF_METRICS__`
- 开启调试输出：
  - `localStorage.setItem('soul-perf-debug', '1')`
  - 刷新页面后查看控制台 `[perf]` 输出
- 关闭调试输出：
  - `localStorage.removeItem('soul-perf-debug')`

## 当前建议关注点

- 图表相关：Dashboard 初次进入时重点关注 `dashboard:initCharts`
- 日历相关：切月和打开日期详情时重点关注 `workCalendar:*`
- 长列表相关：筛选、翻页、排序时重点关注各 `*fetch*Table`
- 如果某个列表页在相同筛选下重复触发多次请求，先检查页面层 watcher、分页事件和自动刷新逻辑

## 结合 chunk 结果的当前判断

- 图表热点仍集中在 `charts-components-vendor` 与 `charts-series-vendor`
- 日期与表格热点仍集中在 `element-plus-date-vendor` 与 `element-plus-table-vendor`
- 大页面主 chunk 仍需重点观察：`UserManagementPage`、`SyncConsolePage`、`MachineScheduleListPage`、`PartScheduleListPage`

## 建议执行方式

1. 运行 `npm run build:observe` 查看构建产物快照
2. 打开 `soul-perf-debug`，手动执行一次筛选 / 翻页 / 切月 / 打开详情
3. 对照 `window.__SOUL_PERF_METRICS__` 和 chunk 快照判断热点是请求、渲染还是依赖体积
