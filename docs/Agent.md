### 2. 执行规则
1. 上述三份文档都是**执行 checklist**，只允许变更复选框状态：`- [ ]` / `- [x]`。
2. 三份文档的**正文内容禁止修改**，包括但不限于：标题、分组、条目描述、验收口径、优先级、说明文字。
3. 不允许在 checklist 正文中追加以下内容：
   - 负责人姓名
   - 临时讨论记录
   - 过程日志
   - SQL 试验结果
   - 口头结论
   - 与勾选无关的备注文字
4. 实际完成情况、验证结果、风险变化、阶段结论，统一写入：
   - `docs/progress/自动排产系统-前端.md`
   - `docs/progress/自动排产系统-后端.md`

### 3. 变更权限
1. 默认只有孔本人明确授权后，才允许修改三份 checklist 的正文。
2. Agent / AI / 开发人员在未获授权时，只能勾选，不得改写条目内容。

### 4. 编码要求
1. 文档必须保持 UTF-8 正常中文。
2. 若出现乱码、连续问号占位符、标题错位或结构异常，必须立即修正。
